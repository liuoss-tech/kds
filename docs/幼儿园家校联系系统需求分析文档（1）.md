## 1. 项目概述

### 1.1 项目背景

旨在构建一个连接幼儿园、教师与家长的数字化家校联系平台，实现幼儿信息管理、生活状态记录、家园互动、考勤请假等功能的在线化，提升管理效率与家园沟通质量。

### 1.2 建设目标

- **管理高效化**：自动化考勤统计、批量账号管理、结构化数据采集。
- **沟通透明化**：家长实时查看幼儿生活状态、通知确认留痕。
- **数据安全化**：严格的权限隔离、敏感信息脱敏、操作日志审计。
- **架构扩展性**：预留视频监控等硬件对接接口，支持未来功能扩展。

### 1.3 适用范围

- **终端适配**：系统需采用**响应式布局**，同时适配电脑端（Web 管理后台）与移动端（手机浏览器/H5）。
- **部署方式**：部署于**幼儿园本地服务器**，数据完全自控。

## 2. 用户角色与权限体系 (RBAC)

系统采用基于角色的访问控制，支持数据范围隔离（本人/本班/全园）。



| 角色           | 数据范围         | 核心权限说明                                                 |
| :------------- | :--------------- | :----------------------------------------------------------- |
| **超级管理员** | 全园数据         | 系统配置、账号管理、权限分配、数据备份、全园报表查看。       |
| **园长**       | 全园数据（只读） | 查看全园数据看板（出勤率、活动参与、通知触达），无日常操作权限。 |
| **班主任**     | 本班全量数据     | 发布通知、审批请假、录入生活记录、导出报表、管理幼儿信息。   |
| **配班老师**   | 本班部分数据     | 仅可录入生活记录/活动照片/考勤，**不可审批/导出/删改幼儿信息**。 |
| **专科老师**   | 任教班级名单     | 仅查看幼儿姓名 + 照片（课堂点名），**不可查看健康/请假等敏感信息**。 |
| **保健医生**   | 全园健康数据     | 录入晨午检体温、标记健康异常、查看全园健康统计报表。         |
| **后勤/食堂**  | 用餐统计数据     | 发布每周食谱、查看各班级用餐人数统计。                       |
| **家长**       | 自家幼儿数据     | **100% 数据隔离**，仅查看绑定子女信息、提交请假、确认通知。  |

## 3. 功能需求详情

### 3.1 账号与基础信息管理

- **账号生成**：由管理员/教师**批量导入**生成初始密码，分发给家长。
- **绑定机制**：建立 `幼儿表 - 家长表` 多对多绑定关系，确保家长只能查看自己孩子信息。
- **信息维护**：支持幼儿基础信息增删改查，教师可修改个人密码及基本信息。
- **敏感信息保护**：界面展示手机号、身份证号时需**部分掩码**（如 `138****1234`）。

### 3.2 幼儿生活状态记录（教师端）

- 记录维度

  （纯结构化表单）：

  1. 🍽️ **饮食情况**：午餐摄入量（选项）、喝水次数（数字）。
  2. 😴 **午睡情况**：入睡时间（时间选择）、睡眠时长（数字）、睡眠质量（枚举：深睡/浅睡/易醒）。
  3. 🚽 **如厕记录**：次数（数字）、异常情况（选项）。
  4. 🌡️ **健康监测**：晨午检体温（数字）、不适症状（选项）。
  5. 📝 **教师备注**：自由文本（用于特殊情况说明）。

- **录入效率**：支持全班统一模板 + 个别修改的批量录入方式。

### 3.3 考勤与请假管理

- **请假流程**：家长提交 → 老师审批 → **系统自动将该幼儿当日考勤标记为“请假”**（无需老师重复操作）。

- **请假类型**：事假、病假、其他（需备注）。

- **证明材料**：**均不强制**上传佐证材料。

- 时间规则

  ：

  - 支持提交未来日期。
  - **支持补交过去日期**（如事后补假）。
  - **天数限制**：单次请假 ≤ 7 天；**超过 7 天需管理员额外审批**（二级审批流）。

- **考勤报表**：支持生成班级维度日报/月报，以及全园汇总报表。

### 3.4 通知与互动管理

- **通知发布**：支持普通通知、**置顶通知**、**强提醒通知**（如紧急停课）。
- **已读判定**：家长需**手动点击“确认收到”按钮**才算已读。
- **未读提醒**：系统自动发送**站内消息**提醒未读家长。
- **私信沟通**：支持家长与教师双向私信。
- **活动管理**：发布活动详情，家长在线报名，教师统计参与人数。

### 3.5 教学与后勤管理

- **教学计划**：发布班级教学计划、上传教学活动记录（图片/文字）。
- **食谱管理**：每日发布班级食谱，后勤可查看用餐人数统计。

## 4. 数据统计与报表

- 统计维度

  ：

  - **出勤统计**：班级日报/月报 + 全园汇总（出勤率排名、趋势图）。
  - **生活记录统计**：人均喝水杯数、午睡达标率、体温异常预警次数。
  - **活动参与统计**：各家庭本学期参与活动次数、类型偏好。

- **导出格式**：仅支持 **Excel (.xlsx)** 格式，便于二次编辑。

- **生成方式**：全部由教师/管理员**手动触发导出**，无定时自动推送任务。

- **权限控制**：导出接口需增加权限校验，防止越权导出全园数据。

## 5. 非功能需求与安全策略

### 5.1 数据安全

- **权限隔离**：教师仅查看任教班级，家长 100% 隔离，接口层增加越权校验。
- **操作日志**：记录关键操作（查看/导出健康记录、修改幼儿信息、审批请假等），包含 `操作人`、`时间`、`IP`、`动作`。
- **删除策略**：高风险操作（删除幼儿、批量导入）需**弹窗二次确认**；数据库建议采用**逻辑删除**（`is_deleted`），但前端不提供撤销恢复功能。

### 5.2 系统异常处理

- **网络异常**：提交表单时若网络中断，直接报错提示“网络异常，请重试”。

- **重复提交**：前端按钮防抖 + 后端唯一索引校验，防止数据重复。

- 文件存储

  ：

  - 存储位置：**服务器本地磁盘**。
  - 限制：单文件 ≤ 10MB，仅允许 jpg/png/mp4。
  - 安全：需配置访问权限控制，防止 URL 遍历访问。

### 5.3 性能与部署

- **用户规模**：约 50 个班级，150 名教师，150 名家长（*注：建议核实家长账号数量，若为 1500 名需调整配置*）。
- **服务器配置建议**：本地服务器最低配置 **4 核 8G + 500GB SSD + RAID1 冗余**。
- **网络要求**：配置路由器端口映射 + DDNS 域名解析，强制开启 **HTTPS 加密**。
- **备份策略**：数据库与文件需**每日自动备份**，备份文件建议异地存储。

### 5.4 扩展性预留

- **接口预留**：架构设计需预留**视频监控对接接口**（以便未来接入园区摄像头）。

- 技术栈

  ：

  - 后端：Java (Spring Boot) + MyBatis Plus
  - 前端：Vue.js + Element UI (响应式布局)
  - 数据库：MySQL 8.0

## 6. 交付物与验收标准

### 6.1 验收标准

- **核心功能**：请假、通知、生活记录等核心流程 **100% 可用**。
- **Bug 容忍度**：允许少量 UI 瑕疵；**不允许**存在数据丢失、权限越权等严重 Bug。
- **测试报告**：需提供核心功能测试报告。

### 6.2 交付文档清单

1. **用户操作手册**（教师版/家长版，含截图步骤）
2. **管理员维护手册**（账号管理、备份恢复流程）
3. **数据库设计文档**（ER 图、数据字典）
4. **API 接口文档**（方便后续二次开发，建议集成 Swagger）
5. **部署安装手册**（本地服务器环境配置步骤）

### 6.3 工期与维护

- **开发周期**：**1 个月**。
- **维护模式**：系统上线后，需约定免费维护期（建议 3 个月），严重 Bug 需 24 小时内修复。

## 7. 风险提示与建议

1. **家长账号数量**：当前预估 150 名家长账号对应 50 个班级（平均每班 3 名家长），若实际为每幼儿 2 名家长（约 3000 名账号），需重新评估服务器并发配置。
2. **本地部署运维**：本地服务器需专人负责运维（备份检查、安全补丁），建议配置 UPS 不间断电源防止断电损坏数据。
3. **文件备份**：由于文件存储在本地磁盘，务必实施“本地 + 异地”双重备份策略，防止硬件故障导致照片丢失。
