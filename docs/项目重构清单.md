# 项目重构清单

> **说明**：本文档基于对仓库当前状态的全面浏览与检索生成，用于记录**建议重构或改进**的项，便于排期与跟踪。  
> **检查范围**：`demo/`（Spring Boot 后端）、`frontend/`（Vue 3 前端）；不包含 `node_modules`、`target` 等生成目录。  
> **更新日期**：2026-03-22  
>
> **当前排期**：**暂不处理「安全与配置」类项**（下表 **SEC-***、**CFG-***），正文仍保留备查，需要时再启用排期。

## 一、优先级约定

| 标签 | 含义 |
|------|------|
| **P0** | 关键路径或高风险，建议尽快处理 |
| **P1** | 可维护性/一致性差，中期应处理 |
| **P2** | 体验或工程化优化，可择机处理 |

> 以下条目**当前不纳入重构排期**，仅作记录。

| ID | 优先级 | 问题描述 | 位置 / 线索 | 建议 |
|----|--------|----------|---------------|------|
| SEC-01 | P0 | 数据源账号密码以明文写在配置中 | `demo/src/main/resources/application.yml`（`spring.datasource`） | 使用环境变量、`spring.config.import` + 外部配置，或 Spring Cloud Config；仓库内提供 `application-example.yml`（无真实密钥） |

## 三、后端架构与代码复用（P1 / P2）

| ID | 优先级 | 问题描述 | 位置 / 线索 | 建议 |
|----|--------|----------|---------------|------|
| BE-01 | P1 | 分页列表 + `LambdaQueryWrapper` + 批量查关联 + `BeanUtils` 组装 VO 的模式在多个 Service 中重复 | 如 `BusiClassServiceImpl`、`BusiStudentServiceImpl`、`BusiNoticeServiceImpl`、`BusiActivityServiceImpl` 等 | 抽取小工具：软删条件、`Page` 构建、按 ID 列表批量查并转 `Map`；复杂 VO 可考虑独立 Assembler |
| BE-02 | P2 | 实体与 VO 映射分散、`BeanUtils.copyProperties` 多处使用 | 约 8 个 `*ServiceImpl` 内多次出现 | 引入 MapStruct 或集中 Converter，减少漏字段与重复代码 |
| BE-03 | P2 | `selectBatchIds` 在 MyBatis-Plus 新版本中属过时 API | `LoginHelper`、`BusiStudentServiceImpl`、`BusiNoticeServiceImpl`、`BusiActivityServiceImpl`、`BusiLeaveApplicationServiceImpl` 等 | 按官方迁移指引改为 `listByIds`（或等价推荐 API） |
| BE-04 | P2 | 依赖注入注解混用 | 部分类 `@Autowired`，部分 `@Resource`（如 `LoginHelper`） | 团队统一为一种（常见为构造器注入 + `final` 字段） |
| BE-05 | P2 | Mapper XML 目录配置存在但未见 XML 映射文件 | `application.yml` → `mapper-locations: classpath*:mapper/**/*.xml`；`src/main/resources/mapper` 目录不存在 | 若全程注解 SQL，可删除冗余配置；若计划用 XML，补目录与规范 |
| BE-06 | P2 | `CustomDataPermissionHandler` 通过 `ApplicationContext` 延迟取 `LoginHelper` | `config/CustomDataPermissionHandler.java` | 评估是否可改为构造器注入；若因循环依赖，与配置类「允许循环依赖」治理一并处理（见第二节 CFG-03，暂缓） |

| ID | 优先级 | 问题描述 | 位置 / 线索 | 建议 |
|----|--------|----------|---------------|------|
| FE-01 | P1 | 生活记录相关 API 拆在两处，风格不一致 | `frontend/src/utils/api.js`（教师端分页/保存）；`frontend/src/api/lifeRecord.js`（家长端） | 合并到 `src/api/lifeRecord.js`（或 `api/busi/lifeRecord.js`），统一命名导出；`LifeRecord.vue` 改引用路径 |

| FE-03 | P2 | 年级展示 `getGradeEmoji` 在多个 Vue 中重复实现 | `Class.vue`、`Student.vue`、`Attendance.vue`、`Recipe.vue`、`LifeRecord.vue`、`AttendanceStat.vue` | 抽成 `utils/grade.js` 或 `composables/useGrade.js` |

## 七、已做得较好的部分（可作为重构时的参照）

- 登录与数据权限相关逻辑集中在 `LoginHelper`、`SaTokenConfig`、`CustomDataPermissionHandler`。
- 前端统一 Axios 实例与响应码处理（`utils/request.js`）。
