# 权限分配模块

> **文档版本**：1.0.0
> **更新时间**：2026-03-18

---

## 一、模块概述

权限分配模块是RBAC权限系统的核心组件，负责将系统菜单权限分配给不同角色。通过树形结构展示权限，支持批量勾选、半选状态处理，实现灵活的权限控制。

## 三、技术实现方案

### 3.1 核心功能点

| 功能 | 实现方法 | 说明 |
|------|----------|------|
| 获取权限树 | `getPermissionTree()` | 递归构建树形权限结构 |
| 获取角色权限 | `getPermissionIdsByRoleId()` | 查询角色当前拥有的权限ID列表 |
| 分配权限 | `assignPermissions()` | 先删后增策略，批量保存权限 |
| 获取我的菜单 | `getMyMenus()` | 根据用户角色动态获取菜单 |
| 首页权限隐藏 | `getPermissionTree()` | 权限分配时隐藏首页相关权限 |
| 首页默认拥有 | `getMyMenus()` | 所有用户默认拥有首页访问权限 |

### 3.2 权限树递归构建算法

```java
@Override
public List<PermissionTreeVO> getPermissionTree() {
    List<SysPermission> allPerms = this.list();
    
    List<PermissionTreeVO> allNodes = allPerms.stream()
        .filter(perm -> perm.getId() != 4L && perm.getId() != 5L)
        .map(perm -> {
            PermissionTreeVO vo = new PermissionTreeVO();
            vo.setId(perm.getId());
            vo.setParentId(perm.getParentId());
            vo.setName(perm.getPermName());
            return vo;
        }).collect(Collectors.toList());
    
    return allNodes.stream()
        .filter(node -> node.getParentId() == 0L)
        .peek(node -> node.setChildren(getChildren(node, allNodes)))
        .collect(Collectors.toList());
}

private List<PermissionTreeVO> getChildren(PermissionTreeVO parent, List<PermissionTreeVO> allNodes) {
    return allNodes.stream()
        .filter(node -> node.getParentId().equals(parent.getId()))
        .peek(node -> node.setChildren(getChildren(node, allNodes)))
        .collect(Collectors.toList());
}
```

### 3.3 权限分配（先删后增策略）

```java
@Override
@Transactional(rollbackFor = Exception.class)
public void assignPermissions(RoleAssignDTO dto) {
    Long roleId = dto.getRoleId();
    List<Long> permissionIds = dto.getPermissionIds();
    
    if (permissionIds == null) {
        permissionIds = new ArrayList<>();
    }
    
    permissionIds.add(4L);
    permissionIds.add(5L);
    
    sysRolePermissionMapper.delete(
        new LambdaQueryWrapper<SysRolePermission>()
            .eq(SysRolePermission::getRoleId, roleId)
    );
    
    if (!permissionIds.isEmpty()) {
        for (Long permId : permissionIds) {
            SysRolePermission rp = new SysRolePermission();
            rp.setRoleId(roleId);
            rp.setPermissionId(permId);
            sysRolePermissionMapper.insert(rp);
        }
    }
}
```

### 3.4 获取我的菜单（首页始终在前）

```java
@Override
public List<SysPermission> getMyMenus(Long userId) {
    List<SysPermission> result = new ArrayList<>();
    
    SysPermission dashboardParent = this.getById(4L);
    if (dashboardParent != null) {
        result.add(dashboardParent);
    }
    SysPermission dashboardChild = this.getById(5L);
    if (dashboardChild != null) {
        result.add(dashboardChild);
    }
    
    List<Long> roleIds = sysUserRoleMapper.selectList(
        new LambdaQueryWrapper<SysUserRole>()
            .eq(SysUserRole::getUserId, userId)
    ).stream().map(SysUserRole::getRoleId).collect(Collectors.toList());
    
    if (!roleIds.isEmpty()) {
        List<Long> permIds = sysRolePermissionMapper.selectList(
            new LambdaQueryWrapper<SysRolePermission>()
                .in(SysRolePermission::getRoleId, roleIds)
        ).stream().map(SysRolePermission::getPermissionId)
         .filter(id -> id != 4L && id != 5L)
         .distinct()
         .collect(Collectors.toList());
        
        if (!permIds.isEmpty()) {
            List<SysPermission> otherPerms = this.listByIds(permIds);
            result.addAll(otherPerms);
        }
    }
    
    return result;
}
```

## 五、数据模型

### 5.1 权限表 (sys_permission)

```sql
CREATE TABLE `sys_permission` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `parent_id` bigint DEFAULT '0' COMMENT '父级ID（0表示顶级）',
  `perm_name` varchar(50) NOT NULL COMMENT '菜单或按钮名称',
  `perm_type` tinyint NOT NULL COMMENT '1-目录 2-菜单 3-按钮',
  `route_path` varchar(100) DEFAULT NULL COMMENT '前端路由路径',
  `api_url` varchar(255) DEFAULT NULL COMMENT '后端接口路径',
  `sort_order` int DEFAULT '0' COMMENT '排序',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  `is_deleted` tinyint DEFAULT '0' COMMENT '逻辑删除',
  PRIMARY KEY (`id`)
);
```

### 5.2 角色权限关联表 (sys_role_permission)

```sql
CREATE TABLE `sys_role_permission` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `role_id` bigint NOT NULL COMMENT '角色ID',
  `permission_id` bigint NOT NULL COMMENT '权限ID',
  PRIMARY KEY (`id`),
  KEY `idx_role_id` (`role_id`),
  KEY `idx_permission_id` (`permission_id`)
);
```

### 6.2 权限分配逻辑

```javascript
async handleAssign(row) {
  this.currentRole = row;
  this.assignDialogVisible = true;
  
  const [treeRes, ownRes] = await Promise.all([
    request.get('/api/system/role/permission/tree'),
    request.get(`/api/system/role/permission/own/${row.id}`)
  ]);
  
  this.permTreeData = treeRes.data;
  this.$nextTick(() => {
    this.$refs.permTree.setCheckedKeys(ownRes.data);
  });
}

async submitAssign() {
  const checkedKeys = this.$refs.permTree.getCheckedKeys();
  const halfCheckedKeys = this.$refs.permTree.getHalfCheckedKeys();
  const allKeys = [...checkedKeys, ...halfCheckedKeys];
  
  await request.post('/api/system/role/permission/assign', {
    roleId: this.currentRole.id,
    permissionIds: allKeys
  });
  
  this.$message.success('权限分配成功！');
  this.assignDialogVisible = false;
}
```

## 八、特殊处理逻辑

### 8.1 首页权限隐藏

权限分配界面不显示"首页看板"和"首页"权限，因为：
- 所有用户都需要访问首页
- 避免误操作导致用户无法访问首页

### 8.2 首页权限自动添加

在 `assignPermissions()` 方法中，自动添加首页权限：
```java
permissionIds.add(4L);  // 首页看板
permissionIds.add(5L);  // 首页
```

### 8.3 半选状态处理

前端提交时需要合并全选和半选节点：
```javascript
const checkedKeys = this.$refs.permTree.getCheckedKeys();
const halfCheckedKeys = this.$refs.permTree.getHalfCheckedKeys();
const allKeys = [...checkedKeys, ...halfCheckedKeys];
```

## 九、安全考虑

1. **事务控制**：权限分配使用 `@Transactional` 保证数据一致性
2. **权限校验**：所有接口需要登录认证
3. **数据隔离**：用户只能看到自己有权限的菜单
4. **操作日志**：建议记录权限变更日志（待实现）
