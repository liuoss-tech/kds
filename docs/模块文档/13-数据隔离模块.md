# 数据隔离模块

> **文档版本**：1.0.0
> **更新时间**：2026-03-18

---

## 一、模块概述

数据隔离模块是幼儿园管理系统的核心安全模块，负责实现不同角色用户之间的数据权限控制，确保：
- 管理员/园长可以查看全园数据
- 老师只能查看分配班级的数据
- 家长只能查看自己绑定的孩子数据

## 三、数据权限设计

### 3.1 角色与数据范围

| 角色 | role_code | data_scope | 数据范围 |
|------|-----------|------------|----------|
| 超级管理员 | SUPER_ADMIN | 1 | 全园数据（无限制） |
| 园长 | PRINCIPAL | 1 | 全园数据（无限制） |
| 班主任 | HEAD_TEACHER | 2 | 分配班级的全部数据 |
| 老师 | TEACHER | 2 | 分配班级的全部数据 |
| 家长 | PARENT | 4 | 绑定孩子的数据 |

### 3.2 data_scope 字段说明

| data_scope 值 | 说明 | 过滤方式 |
|---------------|------|----------|
| 1 | 全园数据 | 无限制 |
| 2 | 本班数据 | WHERE class_id IN (分配的班级IDs) |
| 4 | 家长数据 | WHERE id IN (绑定孩子IDs) |

> **注意**：data_scope 存储在 `sys_role` 表中，每个角色配置不同的数据范围。

## 四、隔离逻辑

### 4.1 老师数据隔离流程

```
1. 用户登录 → 获取用户ID和角色
2. 查询 busi_teacher_class 表，获取老师分配的所有班级ID列表
3. 查询时自动注入: WHERE class_id IN (班级ID列表)
4. 返回过滤后的数据
```

### 4.2 家长数据隔离流程

```
1. 用户登录 → 获取用户ID和角色
2. 查询 busi_student_parent 表，获取家长绑定的所有孩子ID列表
3. 查询时自动注入: WHERE id IN (孩子ID列表)
4. 返回过滤后的数据
```

### 4.3 管理员/园长

```
1. 用户登录 → 获取用户ID和角色
2. data_scope = 1，不进行任何数据过滤
3. 返回全部数据
```

## 六、使用说明

### 6.1 配置角色数据范围

在 `sys_role` 表中设置角色的 `data_scope` 字段：

```sql
-- 超级管理员
UPDATE sys_role SET data_scope = 1 WHERE role_code = 'SUPER_ADMIN';

-- 老师
UPDATE sys_role SET data_scope = 2 WHERE role_code IN ('TEACHER', 'HEAD_TEACHER');

-- 家长
UPDATE sys_role SET data_scope = 4 WHERE role_code = 'PARENT';
```

### 6.2 分配老师到班级

通过 `busi_teacher_class` 表分配老师到班级：

```sql
-- 将用户ID为5的老师分配到班级1
INSERT INTO busi_teacher_class (teacher_user_id, class_id, is_head_teacher) 
VALUES (5, 1, 1);
```

### 6.3 绑定家长与幼儿

通过 `busi_student_parent` 表绑定家长：

```sql
-- 将用户ID为10的家长绑定到幼儿ID为100
INSERT INTO busi_student_parent (student_id, parent_user_id, relation, is_primary) 
VALUES (100, 10, '妈妈', 1);
```

## 七、注意事项

1. **家长权限限制**：家长角色只能查看数据，不能进行增删改操作（需在 Service 层额外判断）

2. **老师多班级**：一个老师可以分配多个班级，查询时会使用 IN 语句

3. **班级多老师**：一个班级可以分配多个老师

4. **家长多孩子**：一个家长可以绑定多个孩子

5. **逻辑删除**：所有业务表均使用逻辑删除（is_deleted 字段）

## 九、扩展说明

### 9.1 添加新的隔离表

如果需要在新的业务表中添加数据隔离，在 `CustomDataPermissionHandler.java` 中添加表名判断：

```java
if ("busi_xxx".equalsIgnoreCase(tableName)) {
    // 根据角色类型返回对应的过滤条件
}
```

### 9.2 自定义过滤逻辑

可以通过重写 `CustomDataPermissionHandler.getSqlSegment()` 方法实现更复杂的过滤逻辑。
