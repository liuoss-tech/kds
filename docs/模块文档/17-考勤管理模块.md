# 考勤管理模块

> **文档版本**：1.0.0
> **更新时间**：2026-03-18

## 1. 模块概述

考勤管理模块是幼儿园家校联系系统的核心模块之一，主要用于管理幼儿每日考勤签到和签退操作，并与请假系统联动自动识别请假幼儿。

### 1.1 功能清单

| 功能 | 说明 |
|:---|:---|
| 考勤列表查询 | 根据班级和日期查询幼儿考勤状态 |
| 签到功能 | 支持单个和批量签到 |
| 签退功能 | 支持单个和批量签退 |
| 请假联动 | 自动识别已审批的请假幼儿并标记 |
| 缺勤标记 | 未签到且无请假的幼儿自动标记为缺勤 |

### 1.2 考勤状态

| 状态值 | 名称 | 说明 |
|:---:|:---|:---|
| 1 | 出勤 | 已签到并签退 |
| 2 | 缺勤 | 既未签到也无请假 |
| 3 | 事假 | 请假（事假） |
| 4 | 病假 | 请假（病假） |
| 5 | 其他请假 | 请假（其他） |

## 3. 业务流程

### 3.1 考勤查询流程

```
1. 教师选择班级和日期（今天或过去）
2. 系统加载班级所有在读幼儿
3. 对每个幼儿检查是否有已审批的请假单
   ├─ 有请假 → 显示请假状态
   └─ 无请假 → 检查是否有考勤记录
      ├─ 有记录 → 显示签到/签退状态
      └─ 无记录 → 显示"未考勤"
```

### 3.2 签到流程

```
1. 教师选择幼儿（单个或批量）
2. 系统检查幼儿是否有已审批的请假
   ├─ 有请假 → 跳过该幼儿
   └─ 无请假 → 记录签到时间和考勤状态为"出勤"
```

### 3.3 签退流程

```
1. 教师选择已签到的幼儿
2. 系统记录签退时间
```

|:---|:---|:---:|:---|
| classId | Long | 是 | 班级ID |
| date | String | 是 | 考勤日期（yyyy-MM-dd） |
| page | Integer | 否 | 页码，默认1 |
| size | Integer | 否 | 每页条数，默认10 |

**响应**

        "signOutTime": "2026-03-18T17:00:00",
        "leaveType": null,
        "leaveTypeText": null,

### 4.3 签到

**请求**

```
POST /api/busi/attendance/sign-in
Content-Type: application/json

{
  "studentIds": [1, 2, 3],
  "classId": 1,
  "date": "2026-03-18"
}
```

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| studentIds | Long[] | 是 | 幼儿ID数组 |
| classId | Long | 是 | 班级ID |
| date | String | 是 | 考勤日期（yyyy-MM-dd） |

**响应**

  "message": "签到成功",
  "data": null

## 7. 业务规则

1. **日期限制**：只能选择今天或过去的日期，明天不可选
2. **请假联动**：自动检查幼儿是否有已审批的请假单，有则显示请假状态
3. **批量跳过**：批量操作时自动跳过已请假的幼儿
4. **数据隔离**：教师只能操作所任班级的幼儿

## 8. 注意事项

1. **表结构复用**：本模块复用数据字典中已有的 `busi_attendance_record` 表
2. **签到时间戳**：系统自动记录当前时间作为签到/签退时间
3. **批量操作**：支持批量签到和签退，提高操作效率
4. **请假优先**：系统优先检查请假状态，请假幼儿不参与签到签退
5. **数据隔离**：各教师只能操作自己所任班级的幼儿
