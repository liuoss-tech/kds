# 幼儿管理模块

## 1. 模块概述

幼儿管理模块是幼儿园家校联系系统的核心模块之一，主要负责管理幼儿档案信息，包括幼儿基本信息、班级分配、家长绑定等功能。

### 1.1 功能清单

| 功能 | 说明 |
|:---|:---|
| 幼儿列表 | 分页展示幼儿信息，支持搜索和筛选 |
| 新增幼儿 | 创建幼儿档案，绑定家长信息 |
| 编辑幼儿 | 修改幼儿信息和家长绑定 |
| 删除幼儿 | 逻辑删除幼儿及家长绑定关系 |
| 家长选择 | 从系统用户中选择家长角色用户 |

## 2. 数据模型

### 2.1 核心表结构

| 表名 | 说明 |
|:---|:---|
| `busi_student` | 幼儿档案表 |
| `busi_student_parent` | 幼儿与家长绑定关系表 |

#### busi_student（幼儿档案表）

| 字段名 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| id | bigint | 是 | 主键ID |
| class_id | bigint | 是 | 所属班级ID |
| student_name | varchar(50) | 是 | 幼儿姓名 |
| gender | tinyint | 否 | 性别：1-男 2-女 0-未知 |
| birthday | date | 否 | 出生日期 |
| id_card | varchar(20) | 否 | 身份证号/护照号（敏感信息） |
| admission_date | date | 否 | 入园日期 |
| status | tinyint | 否 | 在园状态：1-在读 2-毕业 3-退园 |
| avatar | varchar(500) | 否 | 头像URL |
| create_time | datetime | 否 | 创建时间 |
| update_time | datetime | 否 | 更新时间 |
| is_deleted | tinyint | 否 | 逻辑删除：0-未删除 1-已删除 |

#### busi_student_parent（幼儿与家长绑定关系表）

| 字段名 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| id | bigint | 是 | 主键ID |
| student_id | bigint | 是 | 幼儿ID |
| parent_user_id | bigint | 是 | 家长用户ID |
| relation | varchar(20) | 是 | 亲属关系（爸爸、妈妈、爷爷等） |

| create_time | datetime | 否 | 绑定时间 |

## 3. 业务流程

### 3.1 新增幼儿流程

```
1. 填写幼儿基本信息（姓名、性别、出生日期、所属班级）
2. 选择家长（从系统家长用户中选择，可多选）
3. 设置每个家长的亲属关系
4. 设置主监护人（默认第一人）
5. 提交保存
```

### 3.2 家长绑定规则

- 一个幼儿可以绑定**多个家长**
- 每个家长需设置**亲属关系**
- 首位家长默认为**主监护人**（可调整）
- 家长必须是系统中的 **userType=2（家长）** 用户

## 5. 前端页面

### 5.1 路由配置

| 路径 | 组件 | 说明 |
|:---|:---|:---|
| `/busi/student` | Student.vue | 幼儿管理主页面 |

### 5.2 页面功能

- **列表展示**：表格形式展示幼儿信息
- **搜索筛选**：按姓名、班级搜索
- **分页**：支持分页查询
- **新增弹窗**：填写幼儿信息和家长绑定
- **编辑弹窗**：修改幼儿信息和家长绑定
- **删除确认**：二次确认后删除

### 5.3 页面风格

与现有班级管理页面保持一致，采用 Element Plus + 粉色可爱风格。

## 6. 权限与数据隔离

### 6.1 权限要求

需要配置以下权限标识：

| 权限标识 | 说明 |
|:---|:---|
| student:list | 幼儿列表查询 |
| student:add | 新增幼儿 |
| student:edit | 编辑幼儿 |
| student:delete | 删除幼儿 |

### 6.2 数据隔离

| 角色 | 数据范围 |
|:---|:---|
| 超级管理员 | 全园幼儿 |
| 园长 | 全园幼儿（只读） |
| 班主任 | 本班幼儿 |
| 配班老师 | 本班幼儿 |

## 7. 注意事项

1. **必填字段**：姓名、性别、出生日期、所属班级、至少一个家长
2. **默认状态**：新增幼儿时默认状态为"在读"
3. **家长选择**：只显示 userType=2 的系统用户
4. **数据隔离**：教师角色只能操作本班幼儿
5. **敏感信息**：身份证号等敏感信息需脱敏显示
