## 一、模块概述

### 1.1 模块定位

活动管理模块用于教师发布幼儿园活动、家长在线报名参加，增强家园互动。

### 1.2 使用角色

| 角色 | user_type | 权限说明 |
|:---|:---:|:---|
| 班主任 | 1 | 可发布活动、查看本班活动报名情况 |
| 配班老师 | 1 | 可查看本班活动报名情况（不可发布） |
| 家长 | 2 | 可报名参加活动、取消报名 |

## 四、功能说明

### 4.1 活动发布

**流程**：
1. 教师点击"发布活动"
2. 填写活动信息（名称、类型、地点、时间、详情）
3. 选择面向班级（只显示所管班级）
4. 点击"发布活动"
5. 系统写入 busi_activity 表
6. 返回活动列表页

**业务规则**：
- 活动时间必须晚于当前时间
- 报名截止时间必须早于活动时间
- 发布后不可修改、不可删除

### 4.2 活动报名

**流程**：
1. 家长进入活动详情页
2. 选择要参加活动的幼儿
3. 选择参与家长人数
4. 可选填写备注
5. 点击"确认报名"
6. 系统写入 busi_activity_registration 表
7. 页面显示"已报名"状态

**防重复机制**：
- 数据库唯一索引 `uk_activity_student`（activity_id + student_id）防止重复报名
- 后端接口需要处理重复报名异常
- 前端：已报名的幼儿不显示在幼儿选择下拉框中

### 4.3 取消报名

**流程**：
1. 家长在活动详情页点击"取消报名"
2. 弹窗确认"确定取消报名吗？"
3. 点击确认
4. 系统更新 busi_activity_registration 表 status = 2
5. 页面恢复"未报名"状态

**业务规则**：
- 只能在报名截止时间前取消
- 取消后可以重新报名

### 4.4 活动状态自动流转

活动状态根据时间和日期自动计算，无需手动操作：

| 状态 | 条件 |
|:---:|:---|
| 报名中 | 当前时间 < 报名截止时间 且 当前时间 < 活动开始时间 |
| 已截止 | 当前时间 >= 报名截止时间 且 当前时间 < 活动开始时间 |
| 进行中 | 活动开始时间 <= 当前时间 < 活动开始时间+4小时 |
| 已结束 | 活动开始时间+4小时 <= 当前时间 |

## 七、数据隔离

| 角色 | 数据范围 | 过滤逻辑 |
|:---|:---|:---|
| 教师 | 自己发布的活动 | WHERE publisher_id = 当前用户ID |
| 家长 | 孩子所在班级的活动 | WHERE target_class_id IN (孩子所在班级IDs) |

**发布权限**：
- 教师只能向所管班级发布活动
- 班级下拉框只显示 LoginHelper.getClassIds() 返回的班级

**报名权限**：
- 家长只能为自己绑定的孩子报名
- 幼儿下拉框只显示 LoginHelper.getChildIds() 返回的孩子
