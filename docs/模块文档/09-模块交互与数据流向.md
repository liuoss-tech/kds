## 二、核心数据流向

### 2.1 用户登录流程

```
1. 前端输入账号密码 → POST /api/auth/login
2. 后端查询用户 → 校验密码 → 创建Sa-Token会话 → 返回Token
3. 前端存储Token到localStorage → 跳转Dashboard
```

**详细步骤：**

| 步骤 | 前端 | 后端 |
|:---:|:---|:---|
| 1 | 用户输入账号密码 | - |
| 2 | 发起 POST /api/auth/login | - |
| 3 | - | SysUserService.login() 验证账号密码 |
| 4 | - | 查询 sys_user_role 获取角色 |
| 5 | - | 调用 StpUtil.login() 生成Token |
| 6 | 接收响应，存储Token | 返回Token和用户信息 |
| 7 | 跳转首页 | - |

### 2.2 用户管理流程

```
1. 管理员进入用户管理页面
2. 前端请求 /api/system/user/page → 获取用户列表
3. 前端新增/修改/删除用户 → 调用对应接口
4. 后端操作数据库 → 返回结果
```

### 2.3 权限分配流程

```
1. 管理员进入角色管理 → 点击"分配权限"
2. 前端请求 /api/system/role/permission/tree → 获取权限树
3. 前端请求 /api/system/role/permission/own/{roleId} → 获取已有权限
4. 前端勾选权限 → POST /api/system/role/permission/assign
5. 后端先删后增 → 保存到 sys_role_permission 表
```

## 三、模块依赖关系

### 3.1 后端模块依赖

| 模块 | 依赖模块 | 说明 |
|------|----------|------|
| AuthController | SysUserService | 登录认证依赖用户服务 |
| UserController | SysUserService | 用户管理依赖用户服务 |
| SysRoleController | SysRoleService、SysPermissionService、SysRolePermissionService | 角色管理需多方协同 |
| SysUserServiceImpl | SysUserRoleMapper、SysRoleMapper | 查询用户关联角色 |
| SysRoleServiceImpl | SysUserRoleMapper | 删除角色时检查用户关联 |
| BusiClassServiceImpl | SysUserService | 查询班主任姓名 |

### 3.2 前端模块依赖

| 模块 | 依赖模块 | 说明 |
|------|----------|------|
| Login.vue | request.js | 发送登录请求 |
| User.vue | request.js | 用户CRUD操作 |
| Role.vue | request.js | 角色和权限操作 |
| router/index.js | localStorage | Token存取 |

## 四、数据流向总结

### 4.1 前端 → 后端

```
前端页面
    ↓
request.js (添加Token)
    ↓
后端接口
    ↓
Controller
    ↓
Service
    ↓
Mapper
    ↓
数据库
```

### 4.2 后端 → 前端

```
数据库
    ↓
Mapper
    ↓
Service (业务处理)
    ↓
Controller (响应封装)
    ↓
request.js (统一处理)
    ↓
前端页面
```

## 五、关键数据转换

### 5.1 实体转VO

在 Service 层使用 BeanUtils.copyProperties 进行转换：

```java
// 实体转VO
UserVO vo = new UserVO();
BeanUtils.copyProperties(user, vo);
```

### 5.2 DTO转实体

在 Controller 接收 DTO 后，在 Service 层转换：

```java
// DTO转实体
SysUser user = new SysUser();
BeanUtils.copyProperties(formDTO, user);
```
