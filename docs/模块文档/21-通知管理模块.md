# 通知管理模块

> **文档版本**：1.0.0
> **更新时间**：2026-03-20

---

## 一、模块概述

通知模块是幼儿园管理系统中连接园方与家庭的重要沟通渠道。模块分为两个子模块：
- **通知管理**：供教师/管理员发布通知，并查看确认情况
- **通知查看**：供家长和教师查看通知并进行确认

---

## 二、权限设计

### 2.1 角色与发布范围

| 角色 | 角色编码 | 发布范围 |
|:---|:---|:---|
| 超级管理员 | SUPER_ADMIN | 全园 + 所有班级 |
| 园长 | PRINCIPAL | 全园 + 所有班级 |
| 班主任 | HEAD_TEACHER | 本班 |
| 配班老师 | ASSISTANT_TEACHER | 本班 |

### 2.2 确认权限

| 角色 | 是否需要确认 | 说明 |
|:---|:---|:---|
| 家长 | 是 | 确认本人子女的通知 |
| 教师 | 是 | 自动过滤本人发布的通知 |

## 三、数据表设计

### 3.1 总线通知表 (busi_notice)

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| id | bigint | 主键 |
| title | varchar(100) | 通知标题 |
| content | text | 通知正文 |
| notice_type | tinyint | 类型：1-日常 2-缴费 3-放假 |
| notice_level | tinyint | 级别：1-普通 2-置顶 3-强提醒 |
| target_scope | tinyint | 范围：1-全园 2-指定班级 |
| target_class_id | bigint | 目标班级ID |
| publisher_id | bigint | 发布人ID |
| publish_time | datetime | 发布时间 |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |
| is_deleted | tinyint | 逻辑删除 |

### 3.2 通知确认表 (busi_notice_receipt)

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| id | bigint | 主键 |
| notice_id | bigint | 通知ID |
| student_id | bigint | 幼儿ID |
| parent_user_id | bigint | 确认的家长用户ID |
| is_confirmed | tinyint | 是否已确认：0-否 1-是 |
| confirm_time | datetime | 确认时间 |

### 3.3 确认逻辑

- 以**幼儿**为维度存储确认记录
- 同一幼儿多个家长，任何一人确认即视为该幼儿已确认
- `parent_user_id` 记录实际点击确认的家长

## 四、业务流程

### 4.1 发布通知流程

```
1. 发布者选择通知类型、级别、发送范围
2. 填写通知标题和正文内容
3. 点击发布
4. 系统插入 busi_notice 记录
5. 系统根据发送范围生成对应的 busi_notice_receipt 记录（按幼儿维度）
6. 发布成功，返回通知ID
```

### 4.2 确认通知流程

```
1. 家长/教师查看通知列表
2. 点击某条通知进入详情页
3. 点击"确认已读"按钮
4. 系统更新对应幼儿的 receipt 记录（is_confirmed=1, confirm_time=当前时间）
5. 返回成功，更新页面状态
```

## 五、API 接口

### 5.1 通知管理接口

#### 分页查询通知列表（管理端）

> **接口路径**：`/api/busi/notice/manage/page`
> **请求方法**：`GET`
> **认证要求**：是

| 参数名 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| page | Integer | 否 | 页码，默认1 |
| size | Integer | 否 | 每页条数，默认10 |
| title | String | 否 | 通知标题（模糊查询） |
| noticeType | Integer | 否 | 通知类型 |
| noticeLevel | Integer | 否 | 通知级别 |
| startDate | String | 否 | 开始日期 |
| endDate | String | 否 | 结束日期 |

#### 发布通知

> **接口路径**：`/api/busi/notice`
> **请求方法**：`POST`
> **认证要求**：是

| 参数名 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| title | String | 是 | 通知标题 |
| content | String | 是 | 通知正文 |
| noticeType | Integer | 是 | 通知类型：1-日常 2-缴费 3-放假 |
| noticeLevel | Integer | 是 | 通知级别：1-普通 2-置顶 3-强提醒 |
| targetScope | Integer | 是 | 发送范围：1-全园 2-指定班级 |
| targetClassId | Long | 否 | 目标班级ID（scope=2时必填） |

#### 获取通知详情

> **接口路径**：`/api/busi/notice/{id}`
> **请求方法**：`GET`
> **认证要求**：是

#### 获取确认情况

> **接口路径**：`/api/busi/notice/{id}/receipts`
> **请求方法**：`GET`
> **认证要求**：是

### 5.2 通知查看接口

## 八、数据隔离实现

### 8.1 教师数据隔离

教师发布通知时，只能选择本班（通过 `LoginHelper.getClassIds()` 获取）。

### 8.2 家长数据隔离

家长查看通知时，只能看到子女所在班级和全园的通知（通过 `LoginHelper.getChildIds()` 获取）。

### 8.3 确认情况隔离

教师查看确认情况时，只能看到本班幼儿的确认状态。
