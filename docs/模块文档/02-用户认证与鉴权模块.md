## 三、技术实现方案

### 3.1 后端实现

**认证流程：**
1. 根据用户名查询用户信息
2. 校验账号状态（是否禁用）
3. 密码明文比对（生产环境应使用哈希加密）
4. 查询用户关联角色，获取角色编码
5. 调用 `StpUtil.login(user.getId())` 生成 Sa-Token 会话
6. 返回 Token 及用户基本信息

**核心代码逻辑：**

```java
// SysUserServiceImpl.login() 方法
@Override
public LoginVO login(LoginDTO loginDTO) {
    // 1. 根据用户名查询用户
    SysUser user = this.getOne(new LambdaQueryWrapper<SysUser>()
            .eq(SysUser::getUsername, loginDTO.getUsername())
            .eq(SysUser::getIsDeleted, 0));

    if (user == null) {
        throw new BusinessException("账号或密码错误");
    }

    // 2. 校验账号状态
    if (user.getStatus() == 0) {
        throw new BusinessException("该账号已被禁用，请联系幼儿园管理员");
    }

    // 3. 校验密码
    boolean isPasswordMatch = loginDTO.getPassword().equals(user.getPassword());
    if (!isPasswordMatch) {
        throw new BusinessException("账号或密码错误");
    }

    // 4. 查询用户关联角色
    SysUserRole userRole = sysUserRoleMapper.selectOne(
            new LambdaQueryWrapper<SysUserRole>()
                    .eq(SysUserRole::getUserId, user.getId())
                    .last("LIMIT 1")
    );

    String roleCode = "UNKNOWN";
    if (userRole != null) {
        SysRole role = sysRoleMapper.selectById(userRole.getRoleId());
        if (role != null) {
            roleCode = role.getRoleCode();
        }
    }

    // 5. 创建Sa-Token会话
    StpUtil.login(user.getId());

    // 6. 返回结果
    LoginVO vo = new LoginVO();
    vo.setToken(StpUtil.getTokenValue());
    vo.setUserId(user.getId());
    vo.setRealName(user.getRealName());
    vo.setUserType(user.getUserType());
    vo.setRoleCode(roleCode);

    return vo;
}
```

## 五、请求/响应示例

### 5.1 登录请求

```json
POST /api/auth/login
{
  "username": "13800138000",
  "password": "123456"
}
```

### 5.2 登录响应（成功）

```json
{
  "code": 200,
  "data": {
    "token": "xxxx-xxxx-xxxx-xxxx",
    "userId": 1,
    "realName": "张老师",
    "userType": 1,
    "roleCode": "HEAD_TEACHER"
  }
}
```

### 5.3 登录响应（失败）

```json
{
  "code": 500,
  "message": "账号或密码错误",
  "data": null
}
```

## 六、Sa-Token 配置

```yaml
# application.yml
sa-token:
  token-name: Authorization
  timeout: 2592000          # Token有效期30天
  active-timeout: -1        # 同一账号多地登录不互斥
  is-concurrent: true       # 支持多地登录
  is-share: false           # Token不共享
  token-style: uuid         # UUID格式Token
```

---

## 七、字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `token` | String | Sa-Token 颁发的身份令牌，后续请求需携带 |
| `userId` | Long | 用户唯一标识 |
| `realName` | String | 用户真实姓名（用于前端显示"欢迎，XXX"） |
| `userType` | Integer | 用户类型：1-教职工 2-家长 |
| `roleCode` | String | 角色编码（前端据此渲染左侧菜单） |
