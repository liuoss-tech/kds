# 请假管理模块

> **文档版本**：1.2.0
> **更新时间**：2026-03-18

## 1. 模块概述

请假管理模块是幼儿园家校联系系统的核心模块之一，主要用于管理幼儿请假申请与审批流程。

### 1.1 功能清单

| 功能 | 说明 |
|:---|:---|
| 家长提交请假 | 家长为幼儿提交请假申请 |
| 班主任审批 | 班主任审批本班幼儿请假申请 |
| 管理员查看 | 园长/管理员查看全园请假记录 |
| 状态筛选 | 按状态筛选请假记录 |

### 1.2 请假类型

| 类型 | 值 | 说明 |
|:---|:---:|:---|
| 事假 | 1 | 因个人原因请假 |
| 病假 | 2 | 因身体不适请假 |
| 其他 | 3 | 其他特殊原因请假 |

### 1.3 审批状态

| 状态 | 值 | 说明 |
|:---|:---:|:---|
| 待审批 | 0 | 等待班主任审批 |
| 已通过 | 1 | 班主任已同意 |
| 待园长审批 | 2 | 超7天需园长审批 |
| 园长已通过 | 3 | 园长已同意 |
| 已拒绝 | 4 | 班主任/园长已拒绝 |
| 已撤销 | 5 | 家长已撤销 |

## 3. 业务流程

### 3.1 请假申请流程

```
1. 家长登录系统
2. 选择要请假的幼儿（如绑定多个幼儿）
3. 选择请假类型（事假/病假/其他）
4. 选择开始日期和结束日期
5. 填写请假原因
6. 填写证明材料URL（可选）
7. 提交请假申请
8. 状态变为"待审批"
```

### 3.2 审批流程（当前实现：单级审批）

```
1. 班主任登录系统
2. 查看本班请假申请列表
3. 选择待审批的请假
4. 选择"通过"或"拒绝"
5. 如选择拒绝，需填写审批意见
6. 提交审批结果
7. 状态更新为"已通过"或"已拒绝"
```

> **注意**：当前实现为单级审批（班主任直接审批）。数据字典中已预留二级审批字段（超7天需园长审批），未来可扩展。

## 4. API 接口

### 4.1 接口列表

| 接口 | 方法 | 说明 |
|:---|:---:|:---|
| `/api/busi/leave/add` | POST | 家长提交请假申请 |
| `/api/busi/leave/page` | GET | 家长获取请假列表 |
| `/api/busi/leave/teacher-page` | GET | 班主任获取请假列表 |
| `/api/busi/leave/admin-page` | GET | 管理员获取请假列表 |
| `/api/busi/leave/approve` | POST | 班主任审批请假 |
| `/api/busi/leave/students` | GET | 获取家长绑定的幼儿列表 |

### 4.2 接口详情

#### 4.2.1 提交请假申请

**请求**

```
POST /api/busi/leave/add
Content-Type: application/json

{
  "studentId": 1,
  "leaveType": 2,
  "startDate": "2026-03-20",
  "endDate": "2026-03-21",
  "reason": "感冒发烧",
  "proofUrl": ""
}
```

**响应**

```json
{
  "code": 200,
  "message": "请假申请已提交",
  "data": null
}
```

#### 4.2.2 审批请假

**请求**

```
POST /api/busi/leave/approve
Content-Type: application/json

{
  "id": 1,
  "auditStatus": 1,
  "teacherRemark": ""
}
```

或拒绝时：

```json
{
  "id": 1,
  "auditStatus": 4,
  "teacherRemark": "不符合请假规定"
}
```

**响应**

```json
{
  "code": 200,
  "message": "审批成功",
  "data": null
}
```

## 7. 权限与数据隔离

### 7.1 权限配置

| 权限ID | 权限名称 | 路由路径 | 父级ID |
|:---|:---|:---|:---|
| 10 | 家长请假 | /busi/leave/parent | 6（业务管理） |
| 11 | 请假审批 | /busi/leave/teacher | 6（业务管理） |
| 12 | 全园请假 | /busi/leave/admin | 6（业务管理） |

### 7.2 角色权限分配（基于user_type）

| 角色 | 权限 | 判断方式 |
|:---|:---|:---|
| 超级管理员 | 家长请假、班主任审批、全园请假 | roleCode=SUPER_ADMIN |
| 园长 | 家长请假、班主任审批、全园请假 | roleCode=PRINCIPAL |
| 班主任 | 班主任审批 | userType=1（教职工） |
| 教师 | 班主任审批 | userType=1（教职工） |
| 家长 | 家长请假 | userType=2（家长） |

> **说明**：从 v1.2.0 版本开始，权限判断从基于 roleCode 改为基于 userType，更加清晰：
> - 教职工（userType=1）：可使用请假审批功能
> - 家长（userType=2）：可使用家长请假功能
> - 管理员（roleCode=SUPER_ADMIN/PRINCIPAL）：可查看全园请假

### 7.3 数据隔离

| 角色 | 数据范围 |
|:---|:---|
| 超级管理员/园长 | 全园所有请假记录 |
| 班主任/教师 | 本班请假记录（按class_id过滤） |
| 家长 | 自己孩子请假记录（按student_id过滤） |

数据隔离通过 MyBatis Plus 的 DataPermissionInterceptor 实现，在 SQL 层面自动注入过滤条件。

## 9. 注意事项

1. **表结构复用**：本模块复用数据字典中已有的 `busi_leave_application` 表，避免重复建表
2. **家长权限**：只有家长才能提交请假，且只能为自己绑定的孩子提交
3. **审批权限**：只有教师才能审批请假
4. **审批状态**：只有"待审批"状态的请假才能审批
5. **审批意见**：拒绝请假时必须填写审批意见
6. **数据隔离**：各角色只能看到有权访问的请假记录
7. **日期校验**：结束日期不能早于开始日期
8. **请假天数**：系统自动计算请假天数（结束日期 - 开始日期 + 1）

## 10. 扩展建议

### 10.1 可扩展功能

- **二级审批**：超7天需园长审批（当前表结构已支持）
- **请假附件上传**：支持上传病历、证明等附件（当前仅支持URL）
- **请假通知**：短信/推送通知家长和教师
- **请假统计**：各班请假率统计
- **请假审核记录**：完整审批流程日志

- 与通知系统联动（审批结果通知家长）
