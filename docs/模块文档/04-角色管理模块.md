### 3.2 权限树递归构建算法

```java
// 核心递归算法 - SysPermissionServiceImpl.getPermissionTree()
public List<PermissionTreeVO> getPermissionTree() {
    // 1. 获取所有权限
    List<SysPermission> allPerms = this.list();

    // 2. 转换为树节点
    List<PermissionTreeVO> allNodes = allPerms.stream()
        .map(perm -> {
            PermissionTreeVO vo = new PermissionTreeVO();
            vo.setId(perm.getId());
            vo.setParentId(perm.getParentId());
            vo.setName(perm.getPermName());
            return vo;
        }).collect(Collectors.toList());

    // 3. 递归构建树（过滤父节点为0的根节点）
    return allNodes.stream()
        .filter(node -> node.getParentId() == 0L)
        .peek(node -> node.setChildren(getChildren(node, allNodes)))
        .collect(Collectors.toList());
}

// 递归获取子节点
private List<PermissionTreeVO> getChildren(PermissionTreeVO parent, List<PermissionTreeVO> allNodes) {
    return allNodes.stream()
        .filter(node -> node.getParentId().equals(parent.getId()))
        .peek(node -> node.setChildren(getChildren(node, allNodes)))
        .collect(Collectors.toList());
}
```

## 五、数据模型

### 5.1 角色表 (sys_role)

```sql
CREATE TABLE `sys_role` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `role_name` varchar(50) NOT NULL,
  `role_code` varchar(50) NOT NULL,
  `data_scope` tinyint NOT NULL DEFAULT '4' COMMENT '数据范围',
  `description` varchar(255) DEFAULT NULL,
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  `is_deleted` tinyint DEFAULT '0',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_role_code` (`role_code`)
);
```

### 5.2 权限表 (sys_permission)

```sql
CREATE TABLE `sys_permission` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `parent_id` bigint DEFAULT '0' COMMENT '父级ID',
  `perm_name` varchar(50) NOT NULL COMMENT '菜单或按钮名称',
  `perm_type` tinyint NOT NULL COMMENT '1-目录 2-菜单 3-按钮',
  `route_path` varchar(100) DEFAULT NULL,
  `api_url` varchar(255) DEFAULT NULL,
  `sort_order` int DEFAULT '0',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  `is_deleted` tinyint DEFAULT '0',
  PRIMARY KEY (`id`)
);
```

### 5.3 关联表

- **sys_user_role**：用户-角色关联表
- **sys_role_permission**：角色-权限关联表
