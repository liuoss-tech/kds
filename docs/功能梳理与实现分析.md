# 幼儿园家校管理系统 - 功能梳理与实现分析文档

## 一、项目概述

### 1.1 项目背景
本项目为**幼儿园家校联系系统**（KDS Kindergarten System），旨在为幼儿园提供一套完整的家校沟通与园所管理解决方案。系统采用前后端分离架构，后端基于 Spring Boot 构建，前端采用 Vue 3 + Element Plus 开发。

### 1.2 技术栈总览

| 层级 | 技术选型 | 版本 |
|------|----------|------|
| **后端框架** | Spring Boot | 2.7.18 |
| **持久层框架** | MyBatis Plus | 3.5.5 |
| **认证鉴权** | Sa-Token | 1.37.0 |
| **数据库** | MySQL | 8.0+ |
| **缓存** | Redis | - |
| **前端框架** | Vue 3 | 3.4+ |
| **UI组件库** | Element Plus | 2.5+ |
| **路由** | Vue Router | 4.x |
| **构建工具** | Vite | 5.x |

## 二、已实现功能模块详解

### 2.1 用户认证与鉴权模块

#### 2.1.1 功能描述
提供用户登录、退出登录、Token 认证等核心功能，是整个系统的安全入口。

#### 2.1.2 技术实现方案

**后端实现：**
- 认证服务：`AuthController` 处理登录/退出请求
- 核心逻辑：[SysUserServiceImpl.login()](demo/src/main/java/com/kds/service/impl/SysUserServiceImpl.java#L38-L79)
  1. 根据用户名查询用户信息
  2. 校验账号状态（是否禁用）
  3. 密码明文比对（生产环境应使用哈希加密）
  4. 查询用户关联角色，获取角色编码
  5. 调用 `StpUtil.login(user.getId())` 生成 Sa-Token 会话
  6. 返回 Token 及用户基本信息

**前端实现：**
- 登录页面：[Login.vue](frontend/src/views/Login.vue)
  - 表单验证（账号、密码必填）
  - 记住账号功能（localStorage）
  - 登录成功后存储 Token 及用户信息到 localStorage
  - 带动画的幼儿园主题 UI 设计

**核心 API 接口：**

| 接口路径 | 请求方式 | 功能说明 |
|---------|---------|---------|
| `/api/auth/login` | POST | 用户登录 |
| `/api/auth/logout` | POST | 退出登录 |

**请求/响应示例：**

```json
// 登录请求
POST /api/auth/login
{
  "username": "13800138000",
  "password": "123456"
}

// 登录响应（成功）
{
  "code": 200,
  "data": {
    "token": "xxxx-xxxx-xxxx-xxxx",
    "userId": 1,
    "realName": "张老师",
    "userType": 1,
    "roleCode": "HEAD_TEACHER"
  }
}
```

#### 2.3.3 权限树递归构建算法

```java
// 核心递归算法 - SysPermissionServiceImpl.getPermissionTree()
public List<PermissionTreeVO> getPermissionTree() {
    // 1. 获取所有权限
    List<SysPermission> allPerms = this.list();
    
    // 2. 转换为树节点
    List<PermissionTreeVO> allNodes = allPerms.stream()
        .map(perm -> {
            PermissionTreeVO vo = new PermissionTreeVO();
            vo.setId(perm.getId());
            vo.setParentId(perm.getParentId());
            vo.setName(perm.getPermName());
            return vo;
        }).collect(Collectors.toList());
    
    // 3. 递归构建树（过滤父节点为0的根节点）
    return allNodes.stream()
        .filter(node -> node.getParentId() == 0L)
        .peek(node -> node.setChildren(getChildren(node, allNodes)))
        .collect(Collectors.toList());
}
```

### 4.2 核心数据流向

#### 4.2.1 用户登录流程

```
1. 前端输入账号密码 → POST /api/auth/login
2. 后端查询用户 → 校验密码 → 创建Sa-Token会话 → 返回Token
3. 前端存储Token到localStorage → 跳转Dashboard
```

#### 4.2.2 权限分配流程

```
1. 管理员进入角色管理 → 点击"分配权限"
2. 前端请求 /api/system/role/permission/tree → 获取权限树
3. 前端请求 /api/system/role/permission/own/{roleId} → 获取已有权限
4. 前端勾选权限 → POST /api/system/role/permission/assign
5. 后端先删后增 → 保存到 sys_role_permission 表
```
