# 请假管理模块规格说明书

## 一、模块概述

### 1.1 业务背景
幼儿园家校联系系统中，家长需要为幼儿提交请假申请，班主任负责审批。本模块实现幼儿请假的全流程管理。

### 1.2 技术栈
- 后端：Spring Boot 2.7.18 + Java 17 + MyBatis Plus 3.5.5
- 前端：Vue 3.5.30 + Element Plus 2.13.5
- 数据库：MySQL 8.0+

---

## 二、功能需求

### 2.1 请假类型
系统支持三种请假类型：
- **事假**：因个人原因请假
- **病假**：因身体不适请假
- **其他**：其他特殊原因请假

### 2.2 审批流程
```
家长提交请假 → 班主任审批（通过/拒绝） → 审批完成
```

### 2.3 角色功能

| 角色 | 功能 |
|------|------|
| 家长 | 提交请假、查看请假记录（自己孩子） |
| 班主任 | 查看并审批本班请假申请 |
| 园长/管理员 | 查看全园所有请假申请 |

### 2.4 数据隔离规则

| data_scope | 角色 | 可查看范围 |
|:---:|:---|:---|
| 1 | SUPER_ADMIN/PRINCIPAL | 全园所有请假记录 |
| 2 | HEAD_TEACHER/TEACHER | 本班请假记录 |
| 4 | PARENT | 自己孩子请假记录 |

## 三、API 设计

### 3.1 后端接口

#### 3.1.1 家长端接口

| 接口路径 | 请求方式 | 功能 |
|---------|---------|------|
| `/api/busi/leave/page` | GET | 家长获取请假列表（自己孩子） |
| `/api/busi/leave/add` | POST | 家长提交请假申请 |
| `/api/busi/leave/students` | GET | 获取家长绑定的幼儿列表 |

#### 3.1.2 班主任端接口

| 接口路径 | 请求方式 | 功能 |
|---------|---------|------|
| `/api/busi/leave/teacher-page` | GET | 班主任获取本班请假列表 |
| `/api/busi/leave/approve` | POST | 班主任审批请假 |

#### 3.1.3 园长/管理员端接口

| 接口路径 | 请求方式 | 功能 |
|---------|---------|------|
| `/api/busi/leave/admin-page` | GET | 管理员获取全园请假列表 |

### 3.2 请求/响应格式

#### 3.2.1 新增请假请求
```json
POST /api/busi/leave/add
{
  "studentId": 1,
  "leaveType": "病假",
  "startDate": "2026-03-20",
  "endDate": "2026-03-21",
  "reason": "感冒发烧",
  "remark": "需要服药"
}
```

#### 3.2.2 审批请求
```json
POST /api/busi/leave/approve
{
  "id": 1,
  "status": "approved",
  "rejectReason": ""
}
```

## 四、数据库设计

### 4.1 表结构：busi_leave

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 主键 |
| student_id | BIGINT | 幼儿ID |
| leave_type | VARCHAR(20) | 请假类型（事假/病假/其他） |
| start_date | DATE | 开始日期 |
| end_date | DATE | 结束日期 |
| reason | VARCHAR(500) | 请假原因 |
| remark | VARCHAR(500) | 备注 |
| status | TINYINT | 状态（0-待审批/1-已通过/2-已拒绝） |
| reject_reason | VARCHAR(500) | 拒绝原因 |
| applicant_id | BIGINT | 申请人ID（家长ID） |
| approver_id | BIGINT | 审批人ID（班主任ID） |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

### 4.2 状态说明

| status | 含义 | 可操作 |
|--------|------|--------|
| 0 | 待审批 | 审批 |
| 1 | 已通过 | 无 |
| 2 | 已拒绝 | 无 |

## 六、数据隔离实现

### 6.1 后端实现
利用现有的数据权限机制：
- 在 `busi_leave` 表添加 `class_id` 字段用于数据隔离
- 班主任查询时自动过滤本班数据
- 家长查询时自动过滤自己孩子数据
- 园长/管理员无过滤条件

### 6.2 菜单配置
```
业务管理
├── 班级管理
├── 幼儿管理
└── 请假管理
    ├── 家长请假（家长角色）
    ├── 班主任审批（班主任角色）
    └── 全园请假（管理员角色）
```
