## 一、模块概述

### 1.1 模块定位

食谱管理模块是幼儿园管理系统的核心业务模块之一，负责教职工为班级规划和记录每日食谱信息。

### 1.2 模块范围

| 类型       | 说明                   |
| :------- | :------------------- |
| **本期范围** | 教职工端食谱管理页面（新增、编辑、查看） |
| **后续扩展** | 家长端食谱查看功能（不在本期范围）    |

### 1.3 使用角色

| 角色     | user\_type | 权限说明           |
| :----- | :--------: | :------------- |
| 园长/管理员 |      1     | 可查看/编辑所有班级食谱   |
| 班主任    |      1     | 可查看/编辑所任班级食谱   |
| 配班老师   |      1     | 可查看/编辑所任班级食谱   |
| 保健医生   |      1     | 可查看/编辑所任班级食谱   |
| 家长     |      2     | 不可进入（查看功能独立模块） |

### 1.4 数据范围

根据 `busi_teacher_class` 表，教职工属于哪些班级，就能在哪些班级下管理食谱。

## 三、API 设计

### 3.1 获取食谱详情

> **接口说明**：获取指定班级指定日期的食谱详情。

| 项目       | 内容                 |
| :------- | :----------------- |
| **接口路径** | `/api/busi/recipe` |
| **请求方法** | `GET`              |
| **认证要求** | 是（教职工用户）           |

#### 请求参数

| 参数名     |   类型   |  必填 | 说明               |
| :------ | :----: | :-: | :--------------- |
| classId |  Long  |  是  | 班级ID             |
| date    | String |  是  | 食谱日期（yyyy-MM-dd） |

#### 响应示例

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": 1,
    "classId": 1,
    "className": "大一班",
    "targetDate": "2026-03-20",
    "breakfast": "鸡蛋、牛奶、面包",
    "morningSnack": "水果",
    "lunch": "米饭、红烧肉、青菜、番茄蛋汤",
    "afternoonSnack": "酸奶",
    "dinner": "",
    "publisherId": 10,
    "publisherName": "张老师",
    "createTime": "2026-03-20 08:00:00",
    "updateTime": "2026-03-20 08:30:00"
  }
}
```

#### 字段说明

| 字段             |   类型   | 说明              |
| :------------- | :----: | :-------------- |
| id             |  Long  | 食谱ID（无数据时为null） |
| classId        |  Long  | 班级ID            |
| className      | String | 班级名称            |
| targetDate     | String | 食谱日期            |
| breakfast      | String | 早餐内容            |
| morningSnack   | String | 早点内容            |
| lunch          | String | 午餐内容            |
| afternoonSnack | String | 午点内容            |
| dinner         | String | 晚餐内容            |
| publisherId    |  Long  | 发布人ID           |
| publisherName  | String | 发布人姓名           |
| createTime     | String | 创建时间            |
| updateTime     | String | 更新时间            |

### 3.2 保存食谱

> **接口说明**：保存食谱（新增或更新），根据 classId + targetDate 判断是否已存在。

| 项目       | 内容                 |
| :------- | :----------------- |
| **接口路径** | `/api/busi/recipe` |
| **请求方法** | `POST`             |
| **认证要求** | 是（教职工用户）           |

#### 请求参数

```json
{
  "classId": 1,
  "targetDate": "2026-03-20",
  "breakfast": "鸡蛋、牛奶、面包",
  "morningSnack": "水果",
  "lunch": "米饭、红烧肉、青菜、番茄蛋汤",
  "afternoonSnack": "酸奶",
  "dinner": ""
}
```

| 参数名            |   类型   |  必填 | 说明               |
| :------------- | :----: | :-: | :--------------- |
| classId        |  Long  |  是  | 班级ID             |
| targetDate     | String |  是  | 食谱日期（yyyy-MM-dd） |
| breakfast      | String |  是  | 早餐内容             |
| morningSnack   | String |  否  | 早点内容             |
| lunch          | String |  是  | 午餐内容             |
| afternoonSnack | String |  否  | 午点内容             |
| dinner         | String |  否  | 晚餐内容             |

#### 响应示例

```json
{
  "code": 200,
  "message": "食谱保存成功",
  "data": null
}
```

## 四、数据字典对照

### 4.1 食谱表（busi\_recipe）

| 字段名              | 中文名   |  必填 | 说明                             |
| :--------------- | :---- | :-: | :----------------------------- |

| dinner           | 晚餐    |  否  | 255字符以内                        |

| update\_time     | 更新时间  |  -  | 自动更新                           |

## 五、交互逻辑

### 5.1 页面初始化流程

```
1. 获取当前用户所在班级列表
2. 默认选中第一个班级
3. 默认日期为当天
4. 查询该班当天食谱
5. 有数据 → 回显到各输入框
6. 无数据 → 显示空白表单
```

### 5.2 日期切换流程

```
切换日期 → 检查当前表单是否有未保存内容 → 
  有 → 提示是否放弃更改
  无 → 直接加载新日期食谱
```

### 5.3 保存逻辑

```
点击保存 → 校验必填项（早餐、午餐） →
  校验失败 → 提示错误
  校验成功 → 调用保存API → 
    成功 → 显示成功提示
    失败 → 显示错误提示
```

## 七、数据隔离

### 7.1 隔离策略

| 角色          | data\_scope | 过滤逻辑                          |
| :---------- | :---------: | :---------------------------- |
| 园长/管理员      |      1      | 无限制（可查看所有班级）                  |
| 班主任/老师/保健医生 |      2      | WHERE class\_id IN (分配的班级IDs) |

### 7.2 写入权限

教职工只能为自己所在班级的食谱数据执行写入操作。

## 八、验收标准

1. ✅ 教职工进入页面后，自动加载所任班级的当天食谱
2. ✅ 可切换班级查看/编辑不同班级的食谱
3. ✅ 可切换任意日期（过去或未来）的食谱
4. ✅ 日期导航支持左右箭头切换和日历选择
5. ✅ 早点、晚餐可折叠/展开
6. ✅ 保存时校验早餐、午餐必填
7. ✅ 保存成功后显示成功提示
8. ✅ 数据按 class\_id 隔离
9. ✅ 界面风格与现有页面保持一致（粉色可爱风格）
