## 一、模块概述

### 1.1 模块定位

家长端食谱查看功能是食谱管理模块的扩展，允许家长查看自己孩子所在班级的每日食谱。家长只能查看，不能修改。

### 1.2 与现有模块的关系

| 模块 | 说明 |
| :--- | :--- |
| 食谱管理（教职工端） | [recipe-management](./recipe-management/spec.md) 的教职工端部分，负责食谱的增删改查 |
| **家长端食谱查看** | 本模块，家长只读查看自己孩子班级的食谱 |

### 1.3 使用角色

| 角色 | user_type | 权限说明 |
| :--- | :--------: | :--- |
| 家长 | 2 | 可查看所绑定幼儿班级的食谱（只读） |

### 1.4 数据范围

根据 `busi_student_parent` 表，家长绑定哪些幼儿，就只能查看这些幼儿所在班级的食谱。

## 二、功能清单

### 2.1 核心功能

| 功能 | 说明 |
| :--- | :--- |
| 幼儿选择 | 家长有多个幼儿时，下拉框选择，默认选中第一个 |
| 日期导航 | 支持左右箭头切换上一天/下一天，日历选择器跳转 |
| 今日跳转 | 一键回到当日 |
| 餐次查看 | 查看早餐、早点、午餐、午点、晚餐（可选） |
| 班级信息 | 显示幼儿所在班级名称 |

### 2.2 不提供功能

| 功能 | 原因 |
| :--- | :--- |
| 编辑/保存 | 家长端只读 |
| 切换幼儿班级 | 只能查看绑定幼儿的班级，不可查看其他班级 |

## 四、API 设计

### 4.1 获取食谱详情（家长端）

> **接口说明**：家长查看自己幼儿班级的食谱详情。

| 项目 | 内容 |
| :--- | :--- |
| **接口路径** | `/api/busi/recipe/parent` |
| **请求方法** | `GET` |
| **认证要求** | 是（家长用户） |

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
| :----- | :---: | :--: | :--- |
| studentId | Long | 是 | 幼儿ID |
| date | String | 是 | 食谱日期（yyyy-MM-dd） |

#### 响应示例

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": 1,
    "studentId": 100,
    "studentName": "张小明",
    "classId": 1,
    "className": "大一班",
    "targetDate": "2026-03-22",
    "breakfast": "鸡蛋、牛奶、面包",
    "morningSnack": "水果",
    "lunch": "米饭、红烧肉、青菜、番茄蛋汤",
    "afternoonSnack": "酸奶",
    "dinner": ""
  }
}
```

#### 字段说明

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | Long | 食谱ID（无数据时为null） |
| studentId | Long | 幼儿ID |
| studentName | String | 幼儿姓名 |
| classId | Long | 班级ID |
| className | String | 班级名称 |
| targetDate | String | 食谱日期 |
| breakfast | String | 早餐内容 |
| morningSnack | String | 早点内容 |
| lunch | String | 午餐内容 |
| afternoonSnack | String | 午点内容 |
| dinner | String | 晚餐内容 |

### 4.2 获取家长绑定的幼儿列表

> **接口说明**：获取当前家长绑定的所有幼儿列表，用于下拉选择。

| 项目 | 内容 |
| :--- | :--- |
| **接口路径** | `/api/busi/student/parent-children` |
| **请求方法** | GET |
| **认证要求** | 是（家长用户） |

#### 响应示例

```json
{
  "code": 200,
  "message": "操作成功",
  "data": [
    {
      "studentId": 100,
      "studentName": "张小明",
      "classId": 1,
      "className": "大一班"
    },
    {
      "studentId": 101,
      "studentName": "李小华",
      "classId": 2,
      "className": "大二班"
    }
  ]
}
```

## 五、数据隔离

### 5.1 隔离策略

| 角色 | data_scope | 过滤逻辑 |
| :--- | :---------: | :------- |
| 家长 | 4 | WHERE student_id IN (家长绑定的幼儿IDs) |

### 5.2 安全约束

- 家长只能查询自己绑定幼儿所在班级的食谱
- 后端需校验 studentId 是否属于当前家长
- 校验失败返回 403 禁止访问

## 七、代码复用策略

### 7.1 复用现有组件

| 复用项 | 复用方式 |
| :----- | :------- |
| 餐次卡片样式 | 复用 Recipe.vue 的 CSS 类名 `.meal-card`、`.meal-header` 等 |
| 日期导航组件 | 复用相同的日期导航结构和逻辑（去掉保存按钮） |
| API 接口 | 复用 `getRecipe` 接口，参数改为 studentId |

### 7.2 差异处理

| 差异项 | 处理方式 |
| :----- | :------- |
| 编辑功能 | 家长端移除保存按钮和所有输入框 |
| 幼儿选择器 | 用幼儿下拉替代班级下拉 |
| 餐次展开状态 | 家长端默认全部展开 |
| 空数据显示 | 内容为空时显示"（暂无内容）" |

## 八、验收标准

1. 家长进入页面后，自动加载所绑定第一个幼儿所在班级的当天食谱
2. 可切换幼儿查看不同班级的食谱
3. 可切换任意日期（历史或未来）的食谱
4. 日期导航支持左右箭头切换和日历选择
5. 今日按钮可快速回到当日
6. 所有餐次内容只读，不可编辑
7. 数据按幼儿-班级关系隔离，家长只能看自己孩子的食谱
8. 界面风格与现有页面保持一致（粉色可爱风格）
9. 幼儿下拉显示"幼儿姓名（班级名）"格式
10. 内容为空的餐次显示"（暂无内容）"
