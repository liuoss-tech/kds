## 一、模块概述

### 1.1 模块定位

家长端生活记录查看功能是生活记录模块的扩展，允许家长查看自己孩子每日的生活记录。家长只能查看，不能修改。

### 1.2 与现有模块的关系

| 模块 | 说明 |
| :--- | :--- |
| 生活记录（教职工端） | [life-record](./life-record/spec.md) 的教职工端部分，负责教师录入幼儿生活数据 |
| **家长端生活记录查看** | 本模块，家长只读查看自己孩子的生活记录 |

### 1.3 使用角色

| 角色 | user_type | 权限说明 |
| :--- | :--------: | :--- |
| 家长 | 2 | 可查看所绑定幼儿的生活记录（只读） |

### 1.4 数据范围

根据 `busi_student_parent` 表，家长绑定哪些幼儿，就只能查看这些幼儿的记录。

## 二、功能清单

### 2.1 核心功能

| 功能 | 说明 |
| :--- | :--- |
| 幼儿选择 | 家长有多个幼儿时，下拉框选择，默认选中第一个 |
| 日期导航 | 支持左右箭头切换上一天/下一天，日历选择器跳转 |
| 今日跳转 | 一键回到当日 |
| 记录查看 | 查看饮食、午睡、如厕、健康监测、备注等各项记录 |
| 班级信息 | 显示幼儿所在班级名称 |

### 2.2 不提供功能

| 功能 | 原因 |
| :--- | :--- |
| 编辑/保存 | 家长端只读 |
| 切换幼儿班级 | 只能查看绑定幼儿的记录，不可查看其他幼儿 |
| 批量导出 | 本期不涉及 |

## 四、API 设计

### 4.1 获取生活记录详情（家长端）

> **接口说明**：家长查看自己幼儿的生活记录详情。

| 项目 | 内容 |
| :--- | :--- |
| **接口路径** | `/api/busi/life-record/parent/page` |
| **请求方法** | `GET` |
| **认证要求** | 是（家长用户） |

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
| :----- | :---: | :--: | :--- |
| studentId | Long | 是 | 幼儿ID |
| date | String | 是 | 记录日期（yyyy-MM-dd） |

#### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "studentId": 100,
    "studentName": "张小明",
    "className": "大一班",
    "recordDate": "2026-03-22",
    "lunchIntake": 1,
    "lunchIntakeText": "全吃光",
    "waterCount": 3,
    "sleepStartTime": "12:30:00",
    "sleepDuration": 120,
    "sleepQuality": 1,
    "sleepQualityText": "深睡",
    "toiletCount": 3,
    "toiletAbnormal": "",
    "morningTemp": 36.5,
    "afternoonTemp": 36.8,
    "healthSymptoms": "",
    "teacherRemark": "今日表现良好"
  }
}
```

#### 字段说明

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | Long | 记录ID（无数据时为null） |
| studentId | Long | 幼儿ID |
| studentName | String | 幼儿姓名 |
| className | String | 班级名称 |
| recordDate | String | 记录日期 |
| lunchIntake | Integer | 午餐摄入量（1-4） |
| lunchIntakeText | String | 午餐摄入量文本 |
| waterCount | Integer | 喝水次数 |
| sleepStartTime | String | 入睡时间（HH:mm:ss） |
| sleepDuration | Integer | 睡眠时长（分钟） |
| sleepQuality | Integer | 睡眠质量（1-3） |
| sleepQualityText | String | 睡眠质量文本 |
| toiletCount | Integer | 如厕次数 |
| toiletAbnormal | String | 异常情况 |
| morningTemp | BigDecimal | 晨检体温 |
| afternoonTemp | BigDecimal | 午检体温 |
| healthSymptoms | String | 不适症状 |
| teacherRemark | String | 教师备注 |

## 五、数据隔离

### 5.1 隔离策略

| 角色 | data_scope | 过滤逻辑 |
| :--- | :---------: | :------- |
| 家长 | 4 | WHERE student_id IN (家长绑定的幼儿IDs) |

### 5.2 安全约束

- 家长只能查询自己绑定幼儿的生活记录
- 后端需校验 studentId 是否属于当前家长
- 校验失败返回 403 禁止访问

## 七、代码复用策略

### 7.1 复用现有组件

| 复用项 | 复用方式 |
| :----- | :------- |
| 页面布局 | 参考 ParentRecipe.vue 的控制栏结构和样式 |
| 日期导航组件 | 复用相同的日期导航结构和逻辑 |
| 幼儿选择器 | 复用 `getParentChildren` API |
| 前端样式 | 复用粉色可爱风格 CSS |
| 枚举值映射 | 复用 lunchIntakeText、sleepQualityText 等文本转换逻辑 |

### 7.2 与教师端生活记录的区别

| 差异项 | 教师端 | 家长端 |
| :----- | :----- | :----- |
| 操作权限 | 可编辑 | 只读 |
| 班级选择 | 有（教师有多个班级） | 无（只能看绑定幼儿的班级） |
| 幼儿选择 | 无（按班级显示全部幼儿） | 有（家长可能有多个孩子） |
| 快速填充 | 有 | 无 |
| 分组展示 | 分组表格式展示 | 一体化卡片式展示 |

## 八、验收标准

1. 家长进入页面后，自动加载所绑定第一个幼儿的当天生活记录
2. 可切换幼儿查看不同孩子的记录
3. 可切换任意日期（历史或未来）的记录
4. 日期导航支持左右箭头切换和日历选择
5. 今日按钮可快速回到当日
6. 所有记录内容只读，不可编辑
7. 数据按幼儿关系隔离，家长只能看自己孩子的记录
8. 界面风格与现有页面保持一致（粉色可爱风格）
9. 幼儿下拉显示"幼儿姓名（班级名）"格式
10. 无数据显示"--"，备注为空显示"暂无备注"
11. 午餐摄入量、睡眠质量等枚举字段显示中文文本
